# Enterprise Network Portfolio Roadmap

## Portfolio Objective

This track demonstrates enterprise network architecture thinking across campus, WAN, data center, cloud edge, and operations. The goal is to show how a network is designed, validated, documented, and explained for real technical review.

## Roadmap

| Phase | Focus | Evidence |
|---|---|---|
| Foundation | Addressing plan, VLAN model, routing domains, naming standards | Topology diagram, IP plan, routing table notes |
| WAN Design | OSPF area design, site-to-site routing, failure domains | CML topology, protocol verification, failover notes |
| Service Provider Concepts | MPLS, LDP, VRF-lite, route exchange | Label table, VRF routing table, packet path explanation |
| Resiliency | BFD, route convergence, redundant links, HA thinking | Before/after convergence notes and validation output |
| Operations | Troubleshooting workflow, change record, monitoring points | Incident notes, show-command checklist, rollback plan |

## Architecture Standards

- Every lab starts with a business or operational requirement.
- Every topology includes the control plane, data plane, and failure scenario.
- Every configuration must include validation output, not only final commands.
- Every page should include interview-ready explanation of what the design proves.

## Target Reviewers

- Network engineers reviewing routing and troubleshooting fundamentals.
- Infrastructure managers reviewing design maturity and documentation quality.
- Recruiters looking for evidence beyond certification badges.
